# 太一 / Taiyi 安装使用指南

> 太一（The One）是一个面向确定性生产任务的 Agent OS，核心设计是**治理权与调度权物理隔离**——同时干活又给活签字的模块会有跳过签字的动机，太一从设计上消除这种动机。
>
> 本指南覆盖：环境准备 → 安装 → 配置 → 运行（CLI / HTTP / Web / Docker）→ 接入真实大模型 → 常用运维命令 → 常见问题。

---

## 1. 环境要求

| 项目 | 要求 |
|---|---|
| Python | **≥ 3.11**（已在 3.13 验证） |
| 操作系统 | macOS / Linux（macOS 支持 `sandbox-exec` 内核级隔离） |
| 依赖 | 核心仅需 `PyYAML`；接入真实大模型需 `httpx`（`[live]` 附加项） |
| 可选 | Docker + docker compose（容器化部署）；Node.js（仅在需要重新构建 Web UI 时） |

核心保持零重依赖：默认离线运行，**零 token、零网络**即可跑通完整闭环。

## 2. 安装

### 方式一（推荐）：pipx —— 全局命令 + 自动隔离环境

[pipx](https://pipx.pypa.io) 会把 taiyi 装进独立环境并把 `taiyi` 命令挂到全局，免去手动管 venv，macOS 上体验最好：

```bash
brew install pipx && pipx ensurepath          # 若还没装 pipx

# 基础安装（离线可跑通全流程）
pipx install "git+https://github.com/zachshi-ai/TaiYi-Agent.git"

# 带真实大模型支持（额外装 httpx）
pipx install "taiyi[live] @ git+https://github.com/zachshi-ai/TaiYi-Agent.git"
```

卸载：`pipx uninstall taiyi`。

### 方式二：pip 直接从 GitHub 安装

仓库 [`zachshi-ai/TaiYi-Agent`](https://github.com/zachshi-ai/TaiYi-Agent) 是公开的，无需 clone（建议先建/激活虚拟环境）：

```bash
# 基础安装（离线可跑通全流程）
pip install "git+https://github.com/zachshi-ai/TaiYi-Agent.git"

# 带真实大模型支持（额外装 httpx）
pip install "taiyi[live] @ git+https://github.com/zachshi-ai/TaiYi-Agent.git"
```

卸载：`pip uninstall taiyi`（只删程序，不碰 `state/` 数据与 `taiyi.yaml`）。

> **生成配置**：pip/pipx 只装代码包，不会自动生成 `taiyi.yaml`。装完跑一次 **`taiyi init`** 即可交互式生成（见第 3 节）；不生成也行——taiyi 有一套完整默认值，直接 `taiyi serve` 就能离线启动。

### 方式三：克隆后本地可编辑安装（开发用）

在仓库根目录（`taiyi.example.yaml` 所在处）执行。建议先建虚拟环境：

```bash
python3 -m venv .venv
source .venv/bin/activate
```

按需选择安装方式：

```bash
# 仅核心 + 测试依赖（离线可跑通全流程）
pip install -e ".[dev]"

# 接入真实大模型（额外装 httpx，OpenAI 兼容适配器需要）
pip install -e ".[live]"

# 生产安装（不含开发依赖）
pip install .
```

安装后会注册命令行入口 `taiyi`（等价于 `python3 -m taiyi.cli`）。

### 方式四：macOS 双击 App（面向不碰终端的用户）

打包成一个双击即用的 `Taiyi.app`——启动后自动开服务并打开浏览器 Web UI，目标机器无需装 Python。构建（在源码根目录）：

```bash
bash deploy/macos/build.sh          # → dist/Taiyi.app
bash deploy/macos/build.sh --dmg    # 另出 dist/Taiyi.dmg（拖入 Applications 分发）
open dist/Taiyi.app                 # 运行
```

- 数据写到 `~/Library/Application Support/Taiyi/`（配置、审计日志、记忆、OODA 轨迹）；首次启动自动生成 `taiyi.yaml`（离线默认，可编辑或用 App 内 Config 面板改，重启生效）。
- **未签名**：本机直接能跑；拷到别的 Mac 首次打开会被 Gatekeeper 拦，需**右键→打开**一次。对外正式分发需 Apple Developer ID 签名+公证（脚本与步骤见 [`deploy/macos/README.md`](./deploy/macos/README.md)）。


验证安装：

```bash
taiyi --help
pytest            # 运行测试套件（263 个用例，应全绿）
```

## 3. 配置

### 最快：`taiyi init` 交互式生成

装完直接跑，按提示回车即可（provider / 模型 / key / executor / 鉴权令牌），自动写出 `taiyi.yaml`：

```bash
taiyi init                       # 交互式，生成 ./taiyi.yaml
taiyi init -o /etc/taiyi.yaml    # 指定输出路径
taiyi init -y                    # 非交互：直接写一份默认（离线）配置
taiyi init --force               # 覆盖已存在的文件
```

> 也完全可以**不生成配置**——taiyi 内置整套默认值，直接 `taiyi serve` 即以离线模式启动。

### 或手动：复制模板

```bash
cp taiyi.example.yaml taiyi.yaml   # 克隆源码时可用
```

关键配置项（`taiyi.yaml`）：

| 字段 | 说明 | 默认 |
|---|---|---|
| `base_dir` | 持久化根目录：审计日志、SQLite 记忆、markdown | `./state` |
| `host` / `port` | HTTP 网关绑定地址（容器内用 `0.0.0.0`） | `127.0.0.1` / `8080` |
| `auth_tokens` | Bearer 鉴权令牌列表，空 = 开放（仅限本地） | `[]` |
| `executor` | `mock`（无副作用，安全默认）/ `sandbox`（真实执行，受治理+凭证隔离） | `mock` |
| `sandbox_dir` | sandbox 执行的工作目录 | `./state/sandbox` |
| `external_git_validation` | sandbox Git 任务启用只读 HEAD/身份复验 | `true` |
| `external_git_remote_validation` | push 后通过只读远端查询复验冻结 branch/SHA（需网络/远端凭据） | `false` |
| `external_github_validation` | 通过 GitHub API 复验 branch/SHA 与平台账号归属（需 `gh auth`） | `false` |
| `github_expected_login` | GitHub 应显示的 author/committer 账号；启用上一项时必填 | `null` |
| `mode` | `agent`（ReAct 循环，逐步治理，默认）/ `workflow`（一次性规划后执行） | `agent` |
| `provider` | `offline`（离线）/ `ollama` / `openai_compat` | `offline` |
| `max_rounds` | 校验失败前的 PDCA 纠正轮数 | `1` |
| `rules_dirs` / `scenarios_dirs` / `skills_dirs` | 自定义规则/场景/技能目录，与内置**合并**（同 id 覆盖） | `[]` |

> 所有字段都可用 `TAIYI_*` 环境变量覆盖（env 优先于文件），便于容器与密钥管理。

`executor: mock` 只验证 Harness 行为。只要任务执行了 mock 工具动作，成功终态就是
`SIMULATED`，不会显示为 `COMPLETED`；切换到真实执行器并通过对应外部验真后，才可声明真实完成。

## 4. 运行

### 4.1 CLI —— 跑单个任务

```bash
taiyi run "commit my changes" --config taiyi.yaml
# 场景会被自动匹配；也可用 --scenario 指定
```

若未安装为命令，可用等价形式：

```bash
PYTHONPATH=src python3 -m taiyi.cli run "commit my changes"
```

### 4.2 HTTP 网关（含 Web UI / 指标 / OpenAI 兼容接口）

```bash
taiyi serve --config taiyi.yaml
# 打开 http://127.0.0.1:8080/  使用内置 React Web UI
#   （对话/任务、审批、OODA 复盘、记忆/指标、配置面板）
```

常用参数：`--host` `--port` `--token`（可重复，运行时追加鉴权令牌）`--static-dir`（自定义 Web 构建产物目录，默认 `web/dist`）。

同源暴露的端点：Web UI（`/`）、任务 API、OpenAI 兼容 `chat` 接口、Prometheus `/metrics`。

### 4.3 MCP 服务器（供 Claude Code / Cursor 等调用，仍受治理）

```bash
taiyi mcp --config taiyi.yaml
# 或：PYTHONPATH=src python3 -m taiyi.cli mcp
```

### 4.4 Docker 部署

```bash
docker compose -f deploy/docker-compose.yml up
```

容器默认 `TAIYI_HOST=0.0.0.0`、`TAIYI_PORT=8080`、`TAIYI_BASE_DIR=/data`（数据卷 `taiyi-data`）。挂载自定义配置：

```yaml
# deploy/docker-compose.yml 中取消注释
volumes:
  - ./taiyi.yaml:/etc/taiyi/taiyi.yaml:ro
command: ["serve", "--config", "/etc/taiyi/taiyi.yaml"]
```

## 5. 接入真实大模型

1. 安装 live 附加项：`pip install -e ".[live]"`
2. 在 `taiyi.yaml`（或 Web 配置面板）设置 provider，重启生效。

任选一种，通过同一个 OpenAI 兼容适配器接入：

```yaml
# 本地 Ollama —— 无需 key
provider: ollama
base_url: http://localhost:11434/v1
model: qwen2.5:7b        # 你 `ollama pull` 过的模型
api_key: null            # Ollama 忽略此项

# 或任意 OpenAI 兼容云模型（DeepSeek / 智谱 / Moonshot / OpenAI / SiliconFlow …）
provider: openai_compat
base_url: https://api.deepseek.com/v1
model: deepseek-chat
api_key: sk-...          # 你的 key，写入配置（已 gitignore），重启生效
# 或用 api_key_env 指向存放 key 的环境变量名（优先于 api_key）
```

要开启真实、内核隔离、受治理的执行（macOS）：

```yaml
executor: sandbox
sandbox_backend: local  # 网络 push 需由容器/主机网络策略隔离；sandbox_exec 会拒绝网络
external_git_remote_validation: true
external_github_validation: true
github_expected_login: zachshi-ai
```

关键保证：**模型无法绕过治理**——即便模型提出被提示注入的工具调用，未获许可就不会执行（不消耗 token）。

## 6. 运维与探索命令

### OODA 建议复盘（人审规则/技能补丁）

太一每个任务会自动归档改进建议；重复失败沉淀为永久治理检查，重复劳动沉淀为受门禁的技能。人工审批后才进入只读规则/技能集：

```bash
taiyi review list                 # 列出待审建议
taiyi review approve <id>         # 批准
taiyi review reject <id>          # 拒绝
```

### 分层示例（位于 `research/examples/`）

```bash
python3 research/examples/governance_demo.py      # 治理引擎（含创始案例）
python3 research/examples/sandbox_demo.py         # 一次受治理的真实 git commit
python3 research/examples/multi_agent_demo.py     # 红线否决 + 优先级仲裁
python3 research/examples/iteration_demo.py       # 失败→永久检查；重复→技能
python3 research/examples/observability_demo.py   # 追踪 / 指标 / 结构化日志
python3 research/examples/llm_offline_demo.py     # 被注入的工具调用仍被拒绝
python3 research/examples/mcp_demo.py             # 太一作为 MCP 服务器
```

治理示例的预期输出（红线/需复核判定）：

```
Identity override (founding incident)   -> DENY          authorship.git_identity.no_override
rm -rf /                                -> DENY          safety.recursive_delete.no_critical_path
git push                                -> NEEDS_REVIEW  dev.git.push_needs_review
Refund 200                              -> NEEDS_REVIEW  customer_service.refund.amount_over_threshold
```

## 7. 规则即数据

红线与场景约束存于 `src/taiyi/rules/*.yaml`，可通过 `git diff` 评审、作为 fixture 测试、无需重解析 prompt 即可加载。新增规则是一次经评审的文件变更，而非调度器的运行时调用：

```yaml
id: authorship.git_identity.no_override
domain: authorship
severity: red_line
applies_to: ["shell:git*"]
trigger: pre_execution
check:
  type: deterministic
  match: args_any
  patterns: ["-c user.name=", "-c user.email=", "--author="]
on_fail:
  action: block
  message: "Overriding the git committer/author identity is forbidden."
precedence: 90
owner: platform-security
```

## 8. 常见问题

- **`taiyi: command not found`** —— 未安装入口或未激活虚拟环境。用 `PYTHONPATH=src python3 -m taiyi.cli ...` 代替，或重新 `pip install -e .`。
- **接入云模型报缺少 httpx** —— 执行 `pip install -e ".[live]"`。
- **改了 key/模型不生效** —— 配置需**重启**服务后生效。
- **想先零成本试跑** —— 保持 `provider: offline` + `executor: mock`，完整闭环即可离线运行。
- **数据存哪里** —— `base_dir`（默认 `./state`）下的审计日志、SQLite 记忆等；轨迹跨重启持久化。
