# Git 安全提交质量门禁

## 准入标准
- Skill 必须包含本文件
- 必含 3 个测试用例(覆盖:正常提交 / 覆盖身份 / 推送)
- 所有测试用例必须跑通

## 退出标准(任一触发即废弃)
- 出现真实生产事故的 committer 错配
- 5 次执行成功率 < 95%
- 用户投诉 3 次以上

## 验证项(自动)
- 提交后必须读取 GitHub 端 commit.author.email
- 验证其与本地 `git config user.email` 一致
- 不一致 → 任务回滚 + 报警

## 副作用清单
- 在用户仓库创建 commit(预期)
- 不影响其它分支(确认)

## 升级流程
- 修改必走 PR
- 需要 1 个管理员审批
- 升级前必须在 staging 仓库跑 5 次
